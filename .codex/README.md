# CODEX Project Settings

Эта директория хранит локальные настройки и правила для агента CODEX в данном репозитории.

## Источники настроек и приоритет
1) Локальная конфигурация (читается ПЕРВОЙ):
   - `.codex/config.yaml` — поведение по умолчанию (роль, язык, стиль вывода, порядок чтения).
   - `.codex/permissions.yaml` — политика прав (путь записи, исключения скана, TPCP: ASK и т.д.).
   - Эти файлы носят консультационный характер и не могут отменять защитные правила из `macros.md` (роль OBS/EXE, TPCP и запреты на разрушительные команды).

2) Общие инструкции пользователя:
   - `prompt.md` (в корне) — основной источник ручных указаний и текущих задач. Читается после `.codex/`.

3) Контекст и артефакты сессии (пишутся агентом):
   - `AGENTS/CODEX_CONTEXT.md` — сводка текущей задачи, находок и следующых шагов.
   - `AGENTS/AGENTS.md` — реестр активностей (обновляется только своя строка).
   - `AGENTS/CODEX-OBS/solutions/*` — предложения (proposals) наблюдателя.
   - `AGENTS/CODEX-OBS/logs/*` — журналы OBS-сессий.
   - `AGENTS/CODEX-EXE/*` — планы внедрения (master_plan.md), логи, ACK для внедрённых предложений.

## Как сохраняются вводные о настройках и запросах
- Настройки по умолчанию и права: в `.codex/config.yaml` и `.codex/permissions.yaml`.
- Ручные запросы/задачи пользователя: в `prompt.md`.

## Как поменять поведение
- Редактируйте `.codex/config.yaml` (язык, роль по умолчанию, стиль) и `.codex/permissions.yaml` (пути записи, политика коммитов/пушей, исключения скана).
- Обновляйте `prompt.md` для постановки/изменения задач (высший приоритет ручных указаний после чтения `.codex/`).
